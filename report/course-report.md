# 知序：面向大学生学习闭环的 HarmonyOS 应用设计与实现

## 摘要

本文设计并实现了一款名为“知序”的 HarmonyOS 学习管理应用，目标是把资料整理、任务生成、专注执行和数据复盘组织成统一的移动端工作流。系统采用 ArkTS 与 ArkUI 完成端侧界面和交互，使用 RDB 保存项目、任务、番茄记录和学习资料，使用 Preferences 保存登录态、计时设置和 AI 接入配置，并保留 Spring Boot + MyBatis + MySQL 后端作为课程全栈展示和后续同步扩展基础。

系统包含今日、任务、资料、专注、复盘五个核心页面。学习资料库支持备忘录、链接说明、课件摘录和文件引用，用户可将资料关联到项目，再通过运行时配置的远程模型生成摘要、要点和下一步专注任务。若远程 AI 未启用、API Key 缺失或请求失败，系统只给出状态提示，不生成替代 AI 建议，也不把失败状态保存为资料摘要。

关键词：HarmonyOS；ArkTS；学习资料库；远程 AI；MVVM

## 第一章 绪论

### 1.1 研究背景和意义

大学生在课程学习中通常需要同时处理任务、课件、笔记、论文摘录、番茄专注和阶段复盘。待办工具偏重任务记录，笔记工具偏重资料沉淀，番茄钟偏重时间管理，三者之间缺少顺畅闭环。知序希望把学习资料和专注行动连接起来，使资料可以被提炼为任务，任务可以进入专注流程，专注记录又可以进入复盘统计。

### 1.2 研究内容

1. 设计项目、任务、番茄记录、学习资料、AI 设置和运行时状态模型。
2. 构建今日、任务、资料、专注、复盘五个核心页面。
3. 实现资料录入、文件引用、项目关联、归档恢复、AI 提炼和从资料生成任务。
4. 集成远程 AI 能力，支持 endpoint、model、API Key 运行时配置和模型列表拉取。
5. 保留 Spring Boot + MyBatis + MySQL 后端工程，作为课程全栈展示基础。

## 第二章 开发工具和技术选型

端侧选择 ArkTS + ArkUI，原因是其能直接使用 HarmonyOS 的 Ability、RDB、Preferences、ArkWeb、资源和构建能力。后端选择 Spring Boot + MyBatis + MySQL，便于展示 REST 接口、服务分层和数据库连接。报告与答辩材料使用 Typst 生成 PDF，便于自动目录和统一排版。

## 第三章 系统设计

系统采用 MVVM 思路组织。模型层定义任务、项目、番茄、资料和 AI 相关结构；视图层由 ArkUI 页面组成；Store 层作为页面和数据库服务之间的统一门面。端侧 RDB 保存 projects、tasks、pomodoros、study_resources 四类核心表，Preferences 保存登录、计时和 AI 设置。

AI 模块采用远程模型接入方案。用户在设置界面填写 endpoint、model 和 API Key。加载设置时，API Key 输入框保持为空，只显示“已保存/未配置”状态。请求失败或未配置时，页面只显示状态提示，不生成本地替代建议。

## 第四章 系统实现与效果

今日页展示学习状态、任务入口和远程 AI 建议卡；任务页支持搜索、筛选、完成、删除和恢复；资料页支持资料录入、文件引用、远程 AI 提炼、生成专注任务和归档恢复；专注页支持番茄计时、暂停、完成和中断记录；复盘页展示本周专注、项目分布、热力图、ArkWeb 图表和学习森林。

报告中的实际截图暂时使用占位符替代，后续可替换为登录入口、今日工作台、任务管理、资料库、AI 设置、专注计时和复盘统计等界面截图。

## 第五章 总结与展望

知序完成了资料、任务、专注和复盘的学习闭环，并通过远程 AI 提升资料到行动的转化效率。后续可继续完善后端同步 UI、冲突处理、资料问答、复习计划生成、真机测试和无障碍体验。
