# UI/UX重构与AI集成优化

## Goal

深度重构 Focus App 的前端界面和用户体验，解决当前五个子页面功能拥挤的问题，应用现代设计风格（毛玻璃/极简风），并跑通远程 AI 调用。

## User Value

- 更清晰的信息架构，减少认知负担
- 更有设计感的视觉体验，提升产品完成度
- 可用的 AI 教练功能，增强学习指导能力

## Requirements

### 1. 项目深入分析
- 分析现有 Index.ets（2000+ 行）的结构和功能分布
- 识别五个子页面（登录、今日、任务、专注、复盘、资源）的 UX 问题
- 评估当前视觉设计的优缺点

### 2. UX 问题诊断
- **信息架构问题**：五个功能区挤在一个 Tabs 中，导航层级不清晰
- **视觉密度问题**：卡片、列表、表单混杂，缺少呼吸感
- **交互流程问题**：快速添加、任务详情、专注启动等关键路径是否流畅
- **状态反馈问题**：本地/远程、加载/成功/失败等状态是否清晰

### 3. UI 重构方向
- **信息架构重组**：
  - 考虑是否需要拆分为多个独立页面
  - 优化 Tab 结构，减少单页面承载的功能
  - 明确主次流程，突出核心任务
  
- **视觉设计升级**：
  - 应用毛玻璃效果（backdrop blur）
  - 极简风格：更大的留白、更清晰的层级
  - 统一的圆角、阴影、间距系统
  - 更现代的配色和渐变

- **组件化拆分**：
  - 将 Index.ets 拆分为可维护的组件
  - 提取 TodayTab、TasksTab、FocusTab、ReviewTab、ResourcesTab
  - 复用卡片、列表、表单等基础组件

### 4. AI 集成验证
- 使用提供的远程 API 配置：
  - `ANTHROPIC_BASE_URL`: `https://token-plan-cn.xiaomimimo.com/anthropic`
  - `ANTHROPIC_DEFAULT_HAIKU_MODEL`: `mimo-v2.5-pro`
  - `ANTHROPIC_AUTH_TOKEN`: (已提供)
- 验证 AI Coach 建议生成
- 验证 Study Resources AI 提炼功能
- 确保远程失败时本地降级正常工作

### 5. 文档更新
- 更新 HANDOVER.md 记录重构内容
- 更新设计文档.md 记录新的 UI 架构和设计系统

## Confirmed Decisions

### 1. 信息架构重组 ✓
**三层导航 + 独立流程**

**主 Tab（3个）：**
- **今日** - Hero + 快速添加 + AI Coach + 下一步专注
- **任务** - 任务列表 + 搜索筛选 + 回收站
- **我的** - 复盘统计 + 学习资源库 + 设置

**独立流程（不占 Tab）：**
- **专注页** - 从任务点击后进入全屏计时器
- **资源详情** - 从"我的"进入的二级页面

**理由**：
- 减少主 Tab 数量（5→3），降低认知负担
- "今日"聚焦高频入口，"任务"独立管理，"我的"整合低频功能
- 专注计时器作为独立流程，避免 Tab 切换打断

### 2. 视觉设计方案 ✓
**核心原则：突出设计感**

**毛玻璃效果：**
- 应用范围：今日 Hero、AI Coach、快速添加、任务卡片
- 实现：半透明白色 `#F8FFFFFF` + `blur(20)` backdrop
- 不使用：页面背景、小标签、按钮

**极简风格：**
- **留白**：卡片间距 16px，内边距 20px，顶部安全区 24px
- **层级**：主标题 24-28px 粗体，次要 14-16px，辅助 12px 浅色
- **圆角**：大卡片 16px，中等 12px，按钮 8px
- **阴影**：主卡片 `radius: 12, offsetY: 4`，悬浮 `radius: 20, offsetY: 8`
- **渐变背景**：页面使用微妙渐变 `['#F3F6FB', '#E8EDF5']`

### 3. 组件拆分策略 ✓
**方案 A：渐进式拆分（@Builder）**

**实现方式：**
- 保持 Index.ets 作为主容器
- 使用 @Builder 提取页面级组件：
  - `@Builder TodayTabBuilder()` - 今日页
  - `@Builder TasksTabBuilder()` - 任务页
  - `@Builder ProfileTabBuilder()` - 我的（复盘+资源+设置）
  - `@Builder FocusTimerBuilder()` - 专注计时器（独立流程）

**选择理由：**
- ✅ 改动最小，风险低
- ✅ 不需要处理跨文件状态传递
- ✅ 构建稳定性高（避免 ArkUI 组件拆分的坑）
- ✅ 适合期末交付时间紧的场景

### 4. AI 调用策略 ✓

**AI Coach（Today 页面）：**
- **调用时机**：
  - 手动刷新按钮（主要方式）
  - 任务状态变更后（完成/新增任务）自动调用一次
- **实现细节**：
  - 显示上次建议 + 生成时间
  - 提供"刷新建议"按钮（带 loading 状态）
  - 远程失败时立即降级到本地规则
  - 同一任务状态下缓存 5 分钟
  - 显示"上次更新：X 分钟前"

**Study Resources AI 提炼：**
- **调用时机**：手动触发（用户点击"AI 提炼"按钮）
- **实现细节**：
  - 资源列表显示"未提炼"/"已提炼"状态
  - 提炼结果保存到 RDB，避免重复请求

**频率控制：**
- 请求失败后 30 秒内不重试，直接使用本地规则
- 避免用户频繁刷新导致过度请求

### 5. App 图标设计 ✓
**方案 B：专注状态抽象符号**

**视觉元素：**
- 同心圆 + 中心点（象征专注聚焦）
- 几何抽象，现代感强

**设计风格：**
- 极简线条
- 渐变填充（从 `#4258C9` 到 `#7A60D0`）
- 适配圆角方形 App 图标规范

**符号意义：**
- 更抽象，适合"学习专注"而非仅"番茄钟"
- 品牌延展性强

**应用场景：**
- App 图标（entry/src/main/resources/base/media/app_icon.png）
- 启动页
- Today Hero 可选装饰元素

### 6. 视觉资源生成
- 使用生图 API：`https://sin.ioll.pp.ua`
- Model: `gpt-image-2`
- 策略：生成拼接的 spritesheet，然后本地裁剪为独立资源
- 需要生成的资源：
  - App 图标（1024x1024，然后缩放为多尺寸）
  - 启动页背景（可选）
  - 新的 UI 装饰元素（如果需要）
