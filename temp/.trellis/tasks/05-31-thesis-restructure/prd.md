# Thesis Deep Restructure (6-chapter, NKU LaTeX)

## Goal

将"移动应用开发"课程结课论文深度重构为 6 章结构、正文 8000–10000 字的本科毕业论文风格工程论文，忠实反映"知序/ZhiXu" HarmonyOS 应用的真实实现，满足《南开大学本科毕业论文指导手册》的格式与标准。权威源为 `report/nku-thesis-template-2020/` 下的 LaTeX 工程（`main.tex` + `abstract.tex` + `course-report.tex` + `references.tex` + `acknowledgements.tex` + `nkthesis.bib`）。

## 背景与定位

- 本论文是"移动应用开发"课程结课作业，老师要求按本科毕业论文格式/标准撰写（非真实毕业论文）。
- 评分维度：选题新颖度、完整度、技术新颖/高端程度、页面复杂度、PPT、论文格式规范、是否带后台/数据库及其复杂度。
- 前端必须用 HarmonyOS（已满足），后端可选（项目已有 Spring Boot + MyBatis + MySQL）。
- 项目此前已用 Typst 写过一版、并迁移到南开 LaTeX 模板成 7 章稿；本次按用户决定推倒重组为 6 章。

## 章节结构（已与用户确认：6 章）

1. **序论**：研究背景与意义 / 国内外研究现状（前人工作的效果与不足） / 本文组织结构。
2. **相关技术与开发工具选型**：HarmonyOS/ArkTS/ArkUI/RDB/Preferences/ArkWeb 等背景知识；对开发语言、IDE、框架做优缺点对比并给出选型理由。
3. **系统设计**：以 MVVM（模型 / 视图 / 控制三部分）为主线的总体设计；数据库表结构设计；网络/AI/后端同步边界设计。
4. **系统实现**：关键模块实现、运行效果（截图占位）、关键技术问题与解决（不贴代码）。
5. **系统测试与结果分析**：测试策略、功能测试、构建验证与兼容性观察、结果分析。
6. **总结与展望**：工作总结 + 未来展望（分两段，正经写未来方向）。

外加：中英文摘要与关键词（3–5 个）、自动生成目录、致谢、参考文献。

## 硬性约束

- 正文不得出现源代码块（老师明确要求；最多放附录，且不计字数）。
- 总结/展望分两段，正经面向未来，**不得自嘲程序 low、有 bug**。
- 关键词 3–5 个；摘要先介绍背景引出问题，再说明解决方法与效果。
- 目录、图表/表格序号、参考文献必须**自动生成**（`\tableofcontents` / biblatex）。
- 内容必须**忠实真实代码**：不得虚构功能、用户实验数据、性能数字或未使用的技术；**不写 CVK / Core Vision Kit**。
- 正文不得写入真实 API 密钥、私有 endpoint（含代码中硬编码的 `xiaomimimo` 地址）、Bearer token。
- 中文正文字数目标 8000–10000。
- 运行截图：先用占位图 + 图注预留位置，用户后续替换图片即可编译。

## 需忠实反映的真实实现要点

- **本地优先架构**：端侧 RDB（`focus.db` v3，4 表 projects/tasks/pomodoros/study_resources）+ Preferences。
- **MVVM**：Model = `FocusModels`/`FocusDatabase`；View = ArkUI 页面（`Index`/`FocusSolo`）；ViewModel = `FocusStore`（统一写入路径 + 统计派生）。
- **5 个工作区**：今日 / 任务 / 资料 / 专注 / 复盘。
- **资料闭环**：资料录入 →（可选）远程 AI 提炼摘要/要点 → 生成专注任务 → 番茄执行 → 复盘统计。
- **远程 AI**：兼容 Chat Completions 接口；未启用 / 无 Key / 非 2xx / 空响应 / 不可解析 → 返回**显式状态降级**，不伪造本地 AI 结果。
- **平台能力**：`FocusAbility` 独立专注窗口（Want/AppStorage 交接，singleton 热启动）、本地通知、后台保护、ArkWeb 复盘图表、服务卡片（**诚实说明当前为静态演示数据**）。
- **数据诚实**：首次启动 DB 注入种子演示数据，但复盘统计由 `FocusStore` 基于 RDB **真实计算**（非 H5 假数据）。
- **后端**：Spring Boot + MyBatis + MySQL，含同步接口（`updated_at` 增量 pull/push + `client_request_id` 幂等插入）；但移动端同步 UI 未产品化，定位为全栈展示与未来多端基础。
- **文件导入局限**：`DocumentViewPicker` 仅保存文件 URI + 备注，不解析 PDF/DOCX/PPTX 正文（诚实局限）。
