在移动互联网与智能终端深度普及的背景下，大学生的学习材料日益分散在聊天记录、网盘、浏览器收藏与便签之中，使得从“查找资料”到“开始学习”的认知路径被不断拉长。与此同时，现有移动学习工具大多停留在单点能力：待办工具长于记录任务却无法关联资料，番茄钟工具能维持专注却缺少任务上下文与复盘分析，笔记工具善于沉淀材料却很少驱动下一步行动，通用 AI 助手擅长生成内容却未与本地学习数据打通。如何在弱网乃至离线条件下，将资料、任务、专注与复盘组织为一条连续且可解释的移动学习闭环，是本文要解决的核心问题。

针对上述问题，本文设计并实现了一款名为“知序”（ZhiXu）的 HarmonyOS 学习管理应用。系统遵循本地优先（local-first）的架构思想，以端侧关系型数据库 RDB 作为可信数据源，保证任务、资料、番茄记录与复盘统计在离线环境下完整可用；客户端采用 ArkTS 语言与 ArkUI 声明式框架，按 MVVM 模式划分模型、视图与控制三层，并以 Preferences 分命名空间保存计时、AI 与登录等运行时配置。系统将今日、任务、资料、专注与复盘五项功能组织为统一闭环：学习资料可经远程 AI 提炼为摘要与要点并一键生成专注任务，专注完成后写入本地记录并反哺复盘统计。远程 AI 仅作为增强层，在未配置或服务不可用时给出显式状态提示而不伪造结果；项目另保留基于 Spring Boot、MyBatis 与 MySQL 的可选后端，提供增量同步与幂等写入接口以展示全栈扩展能力。

实现结果表明，该应用能够在纯离线状态下完成“资料整理—任务分解—番茄执行—复盘反馈”的完整学习闭环，客户端 HAP 与后端 JAR 均通过构建验证。系统涵盖 HarmonyOS 端侧开发、本地数据库设计、原生能力调用、AI 能力集成与 RESTful 接口设计等课程知识要点，具有较好的实践价值与可扩展性。
}

\begin{abstract}
\begin{spacing}{1.5}
In the context of ubiquitous mobile Internet and smart devices, college students' learning materials are increasingly scattered across chat logs, cloud drives, browser bookmarks and sticky notes, which lengthens the cognitive path from ``finding materials'' to ``starting to study''. Meanwhile, existing mobile learning tools remain single-purpose: to-do apps record tasks but cannot associate materials, Pomodoro timers sustain focus but lack task context and review analytics, note apps accumulate materials yet rarely drive the next action, and general-purpose AI assistants generate content without connecting to local study data. The core problem addressed in this thesis is how to organize materials, tasks, focus and review into a continuous and explainable mobile learning loop that remains usable under weak-network or even offline conditions.

To address this problem, this thesis designs and implements ZhiXu, a HarmonyOS study-management application that follows a local-first architecture. It uses the on-device relational database (RDB) as the single source of truth, so that tasks, resources, Pomodoro records and review statistics remain fully available offline. The client is built with the ArkTS language and the ArkUI declarative framework, organized into model, view and control layers under the MVVM pattern, while Preferences stores timer, AI and login runtime configurations in separate namespaces. Five functions---Today, Tasks, Resources, Focus and Review---are unified into one loop: a study resource can be distilled by a remote AI into a summary and key points and turned into a focus task with a single tap; once the Pomodoro session completes, it is written locally and feeds back into review statistics. The remote AI serves only as an enhancement layer, returning explicit status messages instead of fabricated results when it is unconfigured or unavailable; an optional backend based on Spring Boot, MyBatis and MySQL provides incremental synchronization and idempotent writes to demonstrate full-stack extensibility.

Experimental results show that the application completes the full ``organize--decompose--focus--review'' learning loop in a purely offline state, and that both the client HAP and the backend JAR pass build verification. The system covers key course topics including HarmonyOS on-device development, local database design, native capability invocation, AI integration and RESTful API design, offering practical value and good extensibility.
\end{spacing}
\end{abstract}
